## 一、课程背景与目标

### 1.1 上节课回顾

上节课我们已经学会了：

- 什么是 **LangChain**（课程中称为 LaunchAI）——一个用于构建大语言模型应用的框架
- 如何在项目中**安装 LangChain 包**
- 如何接入**通义（Tongyi）**大语言模型
- 如何编写第一个 LLM 应用程序，实现向模型提问并获得回答

上节课的核心代码结构回顾：

```python
# 回顾：通义模型的接入方式
from langchain_community.chat_models import ChatTongyi

chat = ChatTongyi(
    api_key="你的通义API_Key",   # ← 通义在创建对象时直接传入 Key
    model="qwen-turbo"
)

messages = [HumanMessage(content="你好，请介绍一下你自己")]
response = chat.invoke(messages)
print(response.content)
```

### 1.2 本节课要解决的问题

| 问题 | 答案预览 |
|------|----------|
| 通义只有一个模型吗？想换模型怎么办？ | LangChain 支持几十种模型供应商，更换很方便 |
| 不同模型的接入方式一样吗？ | 核心思路相同，但细节有差异（Key 设置方式、包名、依赖等） |
| 怎么知道某个模型能不能用？ | 查官网文档 + 确认网络环境 |
| 万一运行报错怎么办？ | 大部分是缺依赖包，`pip install` 就能解决 |

### 1.3 核心认知

> **更换模型的本质就是更换 API 地址和认证方式**，LangChain 已经帮我们把不同供应商的差异封装好了。我们只需要：
> 1. 安装对应的包
> 2. 导入对应的类
> 3. 传入对应的 API Key
> 4. 其余代码几乎不变

## 二、LangChain 的模块化架构

### 2.1 为什么要拆分成多个包？

LangChain 采用**插件化**的架构设计。这意味着：

- **核心框架**（`langchain-core`）：提供统一的接口定义、消息类型、链式调用等基础能力
- **社区集成**（`langchain-community`）：包含大量第三方集成的实现
- **独立供应商包**（如 `langchain-openai`、`langchain-google-genai`）：每个大模型厂商有自己独立的包

```
langchain（核心框架）
    ├── langchain-core        ← 基础抽象、接口定义
    ├── langchain-community   ← 社区维护的第三方集成（包含智谱AI、通义等）
    │       ├── chat_models/  ← 聊天模型
    │       │     ├── ChatTongyi      → 通义千问
    │       │     ├── ChatZhipuAI     → 智谱AI / ChatGLM
    │       │     ├── ChatBaiduQianfan → 百度千帆
    │       │     └── ...
    │       ├── llms/         ← 普通 LLM（非聊天型）
    │       └── document_loaders/ ← 文档加载器（非语言模型！）
    │
    ├── langchain-openai      ← OpenAI 独立包
    ├── langchain-google-genai ← Google Gemini 独立包
    └── ...（更多供应商独立包）
```

### 2.2 为什么有的模型在 community 里，有的有独立包？

| 位置 | 说明 | 示例 |
|------|------|------|
| **独立包**（`langchain-xxx`） | LangChain 官方重点维护，质量高、更新快 | `langchain-openai`、`langchain-google-genai` |
| **社区集成**（`langchain-community`） | 社区贡献维护，覆盖范围更广 | `ChatZhipuAI`、`ChatTongyi`、`ChatBaiduQianfan` |

实际使用时区别不大，导入路径不同而已：

```python
# 独立包方式
from langchain_openai import ChatOpenAI         # pip install langchain-openai

# 社区集成方式
from langchain_community.chat_models import ChatZhipuAI  # 自带在 langchain-community 中
```

### 2.3 安装方式对比

当你安装 `langchain` 时，它会**自动附带安装** `langchain-community`，所以社区集成里的模型（通义、智谱AI 等）不需要额外安装 langchain 层面的包，只需要安装它们自身的底层依赖。

```
安装 langchain → 自动获得 langchain-community
                 → ChatTongyi、ChatZhipuAI、ChatBaiduQianfan 等都可以直接 import
                 
                但它们的底层 SDK 依赖需要单独安装：
                智谱AI → 需要 pyjwt
                OpenAI → 需要 openai 包
                等等...
```

## 三、如何在官网查找模型供应商

### 3.1 路线一：从快速教程入口查找

这是**最推荐**的入门路径，适合第一次接触某个模型的场景。

**操作步骤：**

1. 打开 LangChain 官方文档网站
2. 在首页找到 **「构建简单的 LLM 应用程序」**（Build a Simple LLM Application）
3. 点击进入后，会看到一个多标签页的教程页面
4. 页面上方有一排**标签卡（Tabs）**，每个标签对应一个模型供应商：

```
[ OpenAI ] [ Google AI ] [ Anthropic ] [ Tongyi ] [ ...更多 ]
```

5. 点击不同的标签卡，页面内容会切换为对应模型的：
   - 需要安装什么包
   - 示例代码是怎样的
   - API Key 如何设置

**关键信息获取点：**

```
标签卡 "OpenAI" 的内容：
├── 安装命令：pip install langchain-openai
├── 导入语句：from langchain_openai import ChatOpenAI
├── Key 设置：ChatOpenAI(api_key="sk-...")
└── 使用示例：完整的调用代码

标签卡 "Google AI" 的内容：
├── 安装命令：pip install langchain-google-genai
├── 导入语句：from langchain_google_genai import ChatGoogleGenerativeAI
├── Key 设置：ChatGoogleGenerativeAI(google_api_key="...")
└── 使用示例：完整的调用代码
```

> 💡 **省心技巧**：这个教程页面给出的代码是**经过验证可以直接使用的**。切换模型时，直接找到对应标签卡，复制代码稍作修改即可，不需要自己去"发明"导入路径和类名。

### 3.2 路线二：从左侧导航查找完整列表

当你需要的模型不在快速教程的标签卡中时，走这条路。

**操作步骤：**

1. 在 LangChain 官网左侧导航栏中，找到 **「Integrations」（集成）** 分类
2. 展开后点击 **「大语言模型供应商的集成」**（Chat models / LLMs providers）
3. 页面会列出常用的模型供应商及其对应的模块
4. 如果列表不够全，页面上会有一个 **「查看所有大模型供应商」** 的链接
5. 点击这个链接进入**完整的供应商目录**

全目录页面长这样（模拟）：


## 四、如何区分聊天模型与其他模块

### 4.1 关键概念

> ⚠️ **LangChain 的集成列表中，并非所有模块都是大语言模型！**

LangChain 的集成分为很多类别：

```
LangChain Integrations（集成）分类：

├── Chat Models（聊天模型）           ← 我们要找的！对话式大语言模型
│   特点：支持多轮对话、有消息系统（System/Human/AI）
│   示例：ChatOpenAI、ChatZhipuAI、ChatTongyi
│
├── LLMs（普通语言模型）              ← 文本补全型，非对话型
│   特点：输入纯文本，输出纯文本，无消息系统
│
├── Document Loaders（文档加载器）     ← 不是语言模型！只是工具
│   特点：加载 PDF、网页、数据库等数据
│   示例：BilibiliLoader（哔哩哔哩文档加载器）
│
├── Vector Stores（向量数据库）
├── Embedding Models（嵌入模型）
├── Tools（工具集成）
└── ...（更多分类）
```

### 4.2 实战判断方法

当你点进一个供应商的页面时，如何快速判断它是否提供聊天模型？

**页面内容检查清单：**

```
✅ 有这个 → 是聊天模型供应商：
├── 页面中出现 "Chat" 字样
├── 有 ChatXxx 类供导入
├── 代码示例包含 SystemMessage / HumanMessage
└── 代码示例中有 .invoke() 调用

❌ 没有这些 → 可能是其他类型的模块：
├── 页面标题是 "Document Loader"
├── 只有加载数据的代码，没有消息系统
├── 没有 API Key 配置说明
└── 代码中导入的是 Loader 类而非 Chat 类
```

### 4.3 具体案例：哔哩哔哩不是聊天模型

以视频中提到的「哔哩哔哩」为例，点进去后你会发现：

```python
# 哔哩哔哩在 LangChain 中是一个"文档加载器"
# 它的功能是从 B 站加载视频字幕/信息，不是对话模型！

from langchain_community.document_loaders import BiliBiliLoader

loader = BiliBiliLoader(video_id="BV1xx411c7mD")
docs = loader.load()   # ← 加载的是数据，不是聊天！
```

所以看到 "Bilibili" 不要误以为 B 站推出了大语言模型，它只是一个数据加载工具。

## 五、选择模型的核心考量因素

### 5.1 网络环境：最首要的因素

这是视频中反复强调的**最重要**的考量点。选错网络环境，代码写对了也跑不通。

| 供应商类型 | 典型代表 | 服务器位置 | 网络要求 | 适用场景 |
|------------|----------|------------|----------|----------|
| **国内供应商** | 通义、智谱AI、百度千帆、腾讯混元、Kimi | 中国大陆 | 正常网络即可 | 国内用户首选 |
| **需科学上网** | OpenAI、Google Gemini、Anthropic Claude | 海外 | 需要代理/VPN | 有网络条件的场景 |

> 🚨 **真实案例**：同学写好了 OpenAI 的调用代码，本地运行一直报 `ConnectionError`。排查了半天代码没问题，最后发现是网络不通，没有科学上网。

**判断流程：**

```
我要用这个模型
    ↓
先看它是哪家公司的？
    ↓
国内公司（阿里、百度、智谱、月之暗面...）
    → ✅ 直接可用，不需要科学上网
    ↓
国外公司（OpenAI、Google、Anthropic、Meta...）
    → ⚠️ 需要确认是否有科学上网环境
       ├── 有 → 可以用
       └── 没有 → 换一个国内供应商，功能和效果差不多
```

### 5.2 费用与配额

| 考量点 | 说明 |
|--------|------|
| **免费额度** | 大多数供应商新注册都有免费体验额度（通常几元到几十元） |
| **计费方式** | 按 token 计费，不同模型价格差异较大 |
| **并发限制** | 免费版通常有 QPS（每秒请求数）限制 |

> 💡 学习阶段建议使用**免费额度**，够做实验就行。不要一上来就充值。

### 5.3 模型能力

不同模型擅长的领域不同：

| 模型 | 优势 | 中文能力 |
|------|------|----------|
| 通义千问（Qwen） | 综合能力强，中文优秀 | ⭐⭐⭐⭐⭐ |
| 智谱AI（GLM） | 学术背景强，开源友好 | ⭐⭐⭐⭐⭐ |
| 百度千帆（ERNIE） | 中文理解深入 | ⭐⭐⭐⭐⭐ |
| OpenAI GPT-4 | 逻辑推理顶尖 | ⭐⭐⭐⭐ |
| Google Gemini | 多模态能力强 | ⭐⭐⭐⭐ |

### 5.4 稳定性与文档质量

优先选择：
- 有**清晰的官方文档**的供应商
- LangChain 集成**比较成熟**的（代码示例完善、更新频繁）
- 社区有**较多讨论**的（遇到问题容易搜索到解决方案）

## 七、实战演示 — 接入智谱AI（ChatGLM）

> 选择智谱AI 的原因：
> - ✅ 国内供应商，无需科学上网
> - ✅ 清华大学团队出品，技术实力强
> - ✅ 提供免费体验额度
> - ✅ LangChain 社区集成成熟

### 7.1 完整流程图

```
第一步：注册账号 + 获取 API Key
    登录官网 → 进入控制台 → API Key 管理 → 创建新 Key → 复制保存
    ⏱ 耗时：约 3 分钟

第二步：安装底层依赖
    conda activate 环境名 → pip install pyjwt==2.10.1
    ⏱ 耗时：约 1 分钟

第三步：编写调用代码
    导入模块 → 设置 Key → 创建模型实例 → 构建消息 → 调用
    ⏱ 耗时：约 5 分钟

第四步：运行测试
    python 脚本名.py → 查看输出 → 根据报错调整
    ⏱ 耗时：约 2 分钟
```

### 7.2 第一步：注册并获取 API Key（详细走位）

**7.2.1 进入智谱AI 官网**

访问智谱AI 开放平台官网，你会看到首页有产品介绍、模型展示等。

**7.2.2 注册 / 登录**

- 新用户点击「注册」，支持手机号或微信扫码注册
- 已有账号直接登录
- 视频中老师是**微信扫码登录**的

**7.2.3 进入 API Key 管理页面（三种路径）**

登录后获取 Key 有三种路径，任选其一：

| 路径 | 操作 | 适用情况 |
|------|------|----------|
| **自动跳转** | 登录后系统可能自动跳转到 Key 管理页面 | 运气好的时候 |
| **控制台路径** | 首页 → 顶部导航「控制台」→ 侧边栏「API Keys」 | 最通用的方式 |
| **手动翻找** | 在首页的个人中心或设置中寻找「API 密钥」入口 | 备用方案 |

**具体操作（控制台路径）：**

```
智谱AI 首页
    ↓ 点击顶部导航栏的「控制台」
进入控制台页面
    ↓ 看左侧菜单栏
找到「API Keys」或「API 密钥」选项
    ↓ 点击进入
看到 API Key 管理页面
    ↓ 会列出已有的 Key（如果是新账号则为空）
```

> 💡 如果首页找不到控制台入口，可以在「模型中心」旁边或头像下拉菜单中寻找。

**7.2.4 创建新的 API Key**

1. 点击页面上的 **「创建 Key」** 或 **「添加新的 API Key」** 按钮
2. 在弹出的对话框中输入 Key 的**名称**（自定义，方便自己识别）
   - 如：`agent2`、`test-key`、`学习专用`
   - 名称没有特殊要求，自己能区分即可
3. 点击 **「确定」** 或 **「创建」**
4. 系统会生成一个新的 Key 并显示在页面上
5. ⚠️ **立即点击「复制」按钮**保存 Key！
   - Key 只会在创建时完整显示一次
   - 关闭页面后就再也看不到了（处于安全考虑）
   - 如果忘记保存，只能删掉重新创建

```
┌─────────────────────────────────────┐
│        创建 API Key                  │
│                                     │
│  名称: [ agent2               ]     │
│                                     │
│  生成后: sk-xxxxxxxxxxxxxxxxxxxxx    │
│           ↑ 只显示这一次，赶紧复制！   │
│                                     │
│  [ 复制 ]  [ 确定 ]                  │
└─────────────────────────────────────┘
```

**7.2.5 关于免费额度**

智谱AI 新注册用户通常会获赠一定的免费体验 token，可以用于测试和学习。在控制台的「概览」或「用量统计」页面可以看到剩余额度。

让我们把代码拆开，逐行理解每一部分的作用和原理。

### 8.1 导入部分

```python
import os
```

**作用**：导入 Python 标准库的 `os` 模块，用于操作环境变量。
**为什么需要它**：智谱AI 要求 API Key 通过环境变量传入，`os.environ` 可以让我们在代码中设置环境变量。

```python
from langchain_community.chat_models import ChatZhipuAI
```

**作用**：从 LangChain 社区集成中导入智谱AI 的聊天模型类。
**路径含义**：
- `langchain_community` → 社区维护的集成包（安装 langchain 时自动附带）
- `chat_models` → 聊天模型子模块（与普通 LLM 不同，支持多轮对话）
- `ChatZhipuAI` → 智谱AI 的具体实现类

```python
from langchain.schema import SystemMessage, HumanMessage
```

**作用**：从 LangChain 核心库中导入消息类型。
**为什么不是从 community 导入**：消息类型是 LangChain 核心定义的统一标准，所有模型共用，属于 `langchain-core`。

### 8.2 Key 设置部分

```python
os.environ["ZHIPUAI_API_KEY"] = "你的Key"
```

**这行代码做了什么**：
1. `os.environ` 是当前进程的环境变量字典
2. `["ZHIPUAI_API_KEY"]` 指定了变量名（这个名称是智谱AI SDK 约定的，不能乱改）
3. `= "你的Key"` 把复制的 Key 赋值给它
4. 之后 `ChatZhipuAI` 内部创建时会自动读取这个环境变量

**智谱AI vs 通义的 Key 设置差异（重要！）**：

```python
# 通义：在构造函数中直接传入
chat = ChatTongyi(api_key="sk-xxx")     # Key 作为参数

# 智谱AI：通过环境变量（或特定参数）
os.environ["ZHIPUAI_API_KEY"] = "xxx"   # 先设置环境变量
chat = ChatZhipuAI(model="glm-4")       # 不需要再传 Key
```

> 为什么会不一样？因为不同供应商的 SDK 设计习惯不同。智谱AI 的官方 SDK 设计就是从环境变量读取 Key 的，LangChain 的 `ChatZhipuAI` 只是遵循了原 SDK 的设计。这也是为什么一定要看官方文档——不同模型的 Key 设置方式可能完全不同。

### 8.3 创建模型实例部分

```python
chat = ChatZhipuAI(
    model="glm-4",
    temperature=0.5
)
```

**参数详解：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `model` | `str` | 是 | 指定使用哪个模型，如 `"glm-4"` |
| `temperature` | `float` | 否（默认通常 0.7-1.0） | 控制回答的随机性，详见第十章 |

### 8.4 构建消息部分

```python
messages = [
    SystemMessage(content="你是一个生活助手，请用中文回复。"),
    HumanMessage(content="夏天适合吃什么水果？")
]
```

**为什么是列表？**

消息列表 = 一段对话的完整上下文。模型会按照列表中的顺序理解对话：

```
[SystemMessage]  → 角色设定，告诉模型"你是谁"
[HumanMessage]   → 用户说的第一句话
[AIMessage]      → 模型回复的内容（多轮对话时使用，第一轮不需要）
[HumanMessage]   → 用户说的第二句话
...以此类推
```

### 8.5 调用与输出部分

```python

```

**`invoke()` 方法**：向模型发送消息并等待完整回复。这是一个**同步**调用，会阻塞直到模型返回结果。

```python
print(response.content)
```

- `response` 是一个 `AIMessage` 对象，包含模型返回的所有信息
- `response.content` 只取其中的**文本内容**（最常用的部分）
- `response` 对象还包含其他元数据（如 token 用量、模型名称等），调试时可以 `print(response)` 查看全貌

## 九、消息系统深入理解

### 9.1 三种消息类型的本质

LangChain 中的消息系统是对大语言模型对话的标准抽象。理解它，就能理解所有模型供应商的统一调用方式。

```python
from langchain.schema import SystemMessage, HumanMessage, AIMessage
```

#### SystemMessage（系统消息）— 规则的制定者

```python
SystemMessage(content="你是一个专业翻译，所有回答都要翻译成英文。")
```

**作用**：
- 设定模型的**角色**（助手、翻译、程序员、心理咨询师……）
- 设定模型的**行为规则**（用什么语言回复、回答多长、什么语气）
- 设定**前置条件**（"你只回答法律相关问题"）

**为什么放在第一个？**
- 系统消息相当于对话的"宪法"，放在列表最前面，模型会优先遵守
- 它不是用户说的话，而是开发者对模型的"后台指令"

**实际效果对比：**

```python
# 没有系统消息
messages = [HumanMessage(content="1+1等于几？")]
# → 模型可能用任何语言回复，风格随意

# 有系统消息
messages = [
    SystemMessage(content="你是一个幼儿园老师，请用最简单的话回答。"),
    HumanMessage(content="1+1等于几？")
]
# → 模型会用幼儿园老师的口吻，非常亲切地回答
```

#### HumanMessage（人类消息）— 用户的发言

```python
HumanMessage(content="今天天气怎么样？")
```

**作用**：代表用户向模型提出的问题或说的话。

**可以有多条吗？** 当然可以！这就是多轮对话的基础。用户在每一轮说的话都用一个 `HumanMessage` 表示。

#### AIMessage（AI 消息）— 模型的发言

```python
AIMessage(content="你好！有什么可以帮你的？")
```

**两种用途：**

**用途一：模型的主动问候（可有可无）**

有些聊天应用打开时模型会主动打招呼。在代码中表现为：

```python
messages = [
    AIMessage(content="你好！我是AI助手，有什么可以帮你的？"),
    HumanMessage(content="帮我查一下天气")
]
```

但在实际开发中，这个主动问候通常不需要，所以大多数代码中会省略。视频中老师也是把官方的 `AIMessage` 示例删掉了。

**用途二：多轮对话中的上下文（非常重要！）**

这是 `AIMessage` 最核心的应用场景——保存对话历史：

```python
# 第一轮
messages = [
    SystemMessage(content="你是数学老师。"),
    HumanMessage(content="1+1等于几？")
]
response1 = chat.invoke(messages)
# 模型返回: "1+1等于2"

# 第二轮：把上一轮的对话加到消息列表里
messages.append(response1)                           # ← 模型的回答
messages.append(HumanMessage(content="那2+2呢？"))   # ← 新问题

response2 = chat.invoke(messages)
# 模型知道上下文（前面问了1+1），返回: "1+1是2，2+2就是4"
```

**为什么要把 AIMessage 加回列表？** 因为大语言模型是**无状态**的——每次调用它不会自动记住之前的对话。我们必须在每次请求时，把整段对话历史都发给它，它才能"记住"上下文。

### 9.2 消息列表的完整结构

一个完整的多轮对话消息列表示例：

```python
messages = [
    # 第0层：系统规则（一般在最前面，只出现一次）
    SystemMessage(content="你是资深Python工程师，请用代码示例回答问题。"),

    # 第1轮对话
    HumanMessage(content="Python中列表和元组有什么区别？"),
    AIMessage(content="列表是可变的（mutable），元组是不可变的（immutable）……"),

    # 第2轮对话
    HumanMessage(content="能不能给个代码示例？"),
    AIMessage(content="当然可以！\n\n```python\nmy_list = [1, 2, 3]\nmy_list[0] = 99\n\nmy_tuple = (1, 2, 3)\n# my_tuple[0] = 99  # 这会报错！\n```"),

    # 第3轮对话（新问题）
    HumanMessage(content="那什么时候应该用元组？"),
]
response = chat.invoke(messages)
# 模型能看到前面的完整对话，给出有上下文的回答
```

### 9.3 消息类型速查表

| 消息类型 | 英文 | 发送者 | 用途 | 典型内容 |
|----------|------|--------|------|----------|
| `SystemMessage` | 系统消息 | 开发者 | 角色设定、规则约束 | "你是中文翻译助手" |
| `HumanMessage` | 人类消息 | 用户 | 提出问题、发起对话 | "翻译这句话" |
| `AIMessage` | AI 消息 | 模型 | 回复内容、对话历史 | "这句话的意思是…" |

## 十、Temperature 温度参数详解

### 10.1 什么是 Temperature？

Temperature（温度）是控制模型输出**随机性**的参数。它影响模型在生成每个词时，选择"最优解"还是"有创意但可能不是最优的解"。

```
Temperature = 想象的"发散程度"或"创意度"
```

### 10.2 取值范围的直观理解

| temperature | 风格 | 行为 | 适合场景 |
|:-----------:|------|------|----------|
| **0.0** | 严谨刻板 | 每次回答基本相同，选最可能的词 | 数学计算、代码生成、事实问答 |
| **0.2 ~ 0.4** | 比较严谨 | 回答稳定，有轻微变化 | 翻译、总结、信息提取 |
| **0.5** | 平衡 | 有一定灵活性但不跑偏 | ⭐ 通用对话（推荐默认值） |
| **0.7 ~ 0.8** | 较有创意 | 回答多样，可能给出意外的好答案 | 创意写作、头脑风暴 |
| **1.0** | 天马行空 | 发散性极强，可能答非所问 | 艺术创作、诗歌生成 |

### 10.3 形象比喻

把模型想象成一个学生，你问他「今天天气怎么样」：

```
temperature = 0.0（最死板）
学生：今天气温25°C，晴转多云，东南风3级。
（严格按数据回答，每次都一样）

temperature = 0.5（适中）
学生：今天天气不错，25度左右，晴转多云，适合出门。
（自然地回答，偶尔换个说法）

temperature = 1.0（最发散）
学生：天气让我想起了一首诗，春天的脚步近了...
     对了你知道吗，古代人观测天气是看云识天气的...
（发散到可能跑题）
```

### 10.4 实际代码示例

```python
# 场景一：代码生成 → 需要严谨
chat_code = ChatZhipuAI(model="glm-4", temperature=0.1)
# 生成的代码更规范、更可预测

# 场景二：日常聊天 → 适中即可
chat_chat = ChatZhipuAI(model="glm-4", temperature=0.5)
# 回答自然，不会太机械也不会太跳脱

# 场景三：创意写作 → 需要发散
chat_creative = ChatZhipuAI(model="glm-4", temperature=0.8)
# 每次生成不同的故事，更有创意

# 场景四：数学/逻辑 → 最低温度
chat_math = ChatZhipuAI(model="glm-4", temperature=0.0)
# 确保数学计算准确，不出奇怪的答案
```

### 10.5 常见误区

> ❌ **误区**：temperature 越高，模型越"聪明"
> ✅ **真相**：temperature 只控制随机性，不改变模型的"知识量"或"推理能力"。temperature=0 的模型一样聪明，只是更"听话"。

> ❌ **误区**：temperature 设为 0 就永远不会出错
> ✅ **真相**：temperature=0 减少的是随机性错误，但模型本身的"知识盲区"导致的错误依然可能存在。

## 十一、API Key 安全管理

### 11.1 为什么不能把 Key 硬编码在代码里？

视频中老师反复强调了这一点。具体风险：

| 风险场景 | 后果 |
|----------|------|
| 代码上传到 **GitHub 公开仓库** | 任何人都能看到你的 Key |
| 使用 **AI 代码审查工具**（如 GitHub Copilot、Cursor 等） | Key 可能被上传到 AI 工具的服务器 |
| 代码**分享给同学/同事** | 对方可以直接使用你的 Key |
| **离职/交接**时代码留在公司 | 前任的 Key 可能被后来者误用 |

> 💸 **最坏情况**：别人拿到你的 Key 后，疯狂调用 API，费用全算在你的账户上。有些供应商没有消费上限，可能一夜之间扣掉大量余额。

### 11.2 安全等级对比

| 级别 | 做法 | 安全性 | 适用 |
|:----:|------|:------:|------|
| 🔴 最不安全 | 直接写死在代码里 | ⭐ | 仅个人临时测试 |
| 🟡 勉强可以 | 放在代码开头的变量里，不上传 Git | ⭐⭐ | 本地实验 |
| 🟢 推荐 | 使用 `.env` 文件 + 不加入 Git | ⭐⭐⭐⭐ | 中小项目 |
| 🟢🟢 最安全 | 系统环境变量 + 密钥管理服务 | ⭐⭐⭐⭐⭐ | 生产环境 |

### 11.3 实操：用 `.env` 文件管理 Key

**步骤 1：安装 python-dotenv**

```bash
pip install python-dotenv
```

**步骤 2：创建 `.env` 文件**

在项目根目录新建 `.env` 文件：

```bash
# .env 文件内容
ZHIPUAI_API_KEY=你的Key粘贴在这里
```

**步骤 3：在 `.gitignore` 中忽略 `.env`**

```bash
# .gitignore 中添加
.env
```

这样 `.env` 文件就不会被 Git 追踪，不会被上传。

**步骤 4：代码中读取**

```python
import os
from dotenv import load_dotenv
from langchain_community.chat_models import ChatZhipuAI
from langchain.schema import SystemMessage, HumanMessage

# 加载 .env 文件中的环境变量
load_dotenv()

# 之后 ChatZhipuAI 会自动从环境变量读取 Key
chat = ChatZhipuAI(model="glm-4", temperature=0.5)

messages = [
    SystemMessage(content="你是一个助手，请用中文回复。"),
    HumanMessage(content="你好，请介绍一下自己。")
]

response = chat.invoke(messages)
print(response.content)
```

### 12.1 在代码中设置 vs 在系统中设置

**方式一：代码中设置（临时）**

```python

os.environ["ZHIPUAI_API_KEY"] = "xxx"
# 只在当前 Python 进程运行期间有效
# 程序退出后失效
```

**方式二：系统环境变量（持久）**

```bash
# macOS / Linux 终端中执行
export ZHIPUAI_API_KEY="你的Key"

# 或者写入 shell 配置文件永久生效
echo 'export ZHIPUAI_API_KEY="你的Key"' >> ~/.zshrc
source ~/.zshrc
```

```bash
# Windows CMD 中执行
set ZHIPUAI_API_KEY=你的Key

# 或通过系统设置 → 环境变量 → 新建
```

**方式三：`.env` 文件（推荐 ✅）**

见上一节，结合 `python-dotenv` 使用。

### 13.1 标准操作流程（SOP）

```
┌─────────────────────────────────────────────────────────────────┐
│                   切换模型标准操作流程（SOP）                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Step 1: 文档调研                                                │
│  ├── 去 LangChain 官网快速教程页面，看有没有这个模型的标签卡        │

│       • 导入什么类？（类名和路径）                                 │

│                                                                 │
│  Step 2: 网络确认                                                │

│                                                                 │
│  Step 5: 编写代码                                                │
```

### 13.2 代码模板

以下是一个**通用模板**，只需要替换三个部分就能接入任何模型：

```python
import os

# ============================================================
# 📌 替换点 1：导入对应的模型类
# ============================================================
# 示例：
#   from langchain_community.chat_models import ChatZhipuAI
#   from langchain_community.chat_models import ChatTongyi
#   from langchain_community.chat_models import ChatBaiduQianfan
#   from langchain_openai import ChatOpenAI
#   from langchain_google_genai import ChatGoogleGenerativeAI
from langchain_community.chat_models import ChatXxx  # ← 改这里

from langchain.schema import SystemMessage, HumanMessage

# ============================================================
# 📌 替换点 2：设置 API Key（方式因模型而异）
# ============================================================
# 方式A：环境变量型（智谱AI 等）
os.environ["XXX_API_KEY"] = "你的Key"  # ← 改环境变量名

# 方式B：参数传入型（通义、OpenAI 等）
# api_key = "你的Key"  # ← 准备好 Key 变量

# ============================================================
# 📌 替换点 3：创建模型实例
# ============================================================
chat = ChatXxx(              # ← 改类名
    # api_key=api_key,       # ← 方式B时取消注释
    model="模型名称",         # ← 改模型名
    temperature=0.5
)

# ============================================================
# 以下代码基本不变，所有模型通用
# ============================================================
messages = [
    SystemMessage(content="你是一个助手，请用中文回复。"),
    HumanMessage(content="你的问题写在这里")
]

response = chat.invoke(messages)
print(response.content)
```

## 十四、更多模型接入示例

学会了智谱AI 的接入方式后，用同样的方法可以快速接入其他模型。以下是几个常用模型的接入代码速查。

### 14.1 百度千帆（ERNIE）

```python
# 安装：langchain-community 已自带，需额外安装 qianfan
# pip install qianfan

import os
from langchain_community.chat_models import ChatBaiduQianfan
from langchain.schema import SystemMessage, HumanMessage

# 百度千帆需要两个 Key
os.environ["QIANFAN_AK"] = "你的Access_Key"
os.environ["QIANFAN_SK"] = "你的Secret_Key"

chat = ChatBaiduQianfan(
    model="ERNIE-Bot-4",    # 或者 "ERNIE-Bot-turbo"
    temperature=0.5
)

    SystemMessage(content="你是一个助手，请用中文回复。"),
    HumanMessage(content="介绍一下成都的美食")
]

response = chat.invoke(messages)
print(response.content)
```

> ⚠️ 注意：百度千帆需要 **两个 Key**（AK + SK），比智谱AI 多一个，这也是为什么一定要看官方文档。

### 14.5 Key 设置方式对比总结

| 模型 | Key 设置方式 | 代码表现 |
|------|-------------|----------|
| 通义 (Tongyi) | 构造函数参数 | `ChatTongyi(api_key="xxx")` |
| 智谱AI (ZhipuAI) | 环境变量 | `os.environ["ZHIPUAI_API_KEY"]="xxx"` |
| 百度千帆 (Baidu) | 环境变量（双Key） | `os.environ["QIANFAN_AK"]` + `QIANFAN_SK` |

| Google | 构造函数参数 | `ChatGoogleGenerativeAI(google_api_key="xxx")` |

> 📌 **结论**：同样是 LangChain 的封装，不同模型的 Key 设置方式可能完全不同。没有统一规律，**必须查看每个模型的官方文档**。

## 十五、常见问题与排错指南

### 15.1 错误速查表

| 错误信息 | 错误类型 | 根本原因 | 解决方法 | 优先级 |
|----------|----------|----------|----------|:------:|
| `ModuleNotFoundError: No module named 'jwt'` | 缺少依赖 | 未安装 pyjwt | `pip install pyjwt==2.10.1` | 🔴高 |
| `ModuleNotFoundError: No module named 'langchain_xxx'` | 缺少包 | 未安装对应供应商包 | `pip install langchain-xxx` | 🔴高 |
| `model not found` | 缺少依赖 | 底层 SDK 依赖缺失 | 装缺的底层包（如 pyjwt） | 🔴高 |
| `ConnectionError` / `ConnectTimeout` | 网络不通 | ①国外供应商未科学上网<br>②开了代理但代理不通 | ①换国内供应商<br>②关闭/检查代理 | 🔴高 |
| `Unauthorized` / `401` | 认证失败 | Key 错误或未设置 | ①检查 Key 是否复制完整<br>②检查 Key 设置方式是否正确 | 🔴高 |
| `Invalid API Key` | Key 无效 | Key 过期或被删除 | 去控制台重新生成 | 🟡中 |
| `Rate limit exceeded` | 请求太频繁 | 免费额度用完了 | ①等待一段时间<br>②换一个 Key<br>③充值 | 🟡中 |
| `Model is overloaded` | 模型过载 | 当前使用人数太多 | 等待几秒后重试 | 🟢低 |

### 15.2 排查问题的标准流程

当代码运行报错时，按以下顺序排查：

```
                  运行报错
                      ↓
          ┌───────────────────────┐
          │ 1. 先看错误信息的最后一行 │
          └───────────────────────┘
                      ↓
       错误类型是 ModuleNotFoundError？
           ↓ 是                    ↓ 否
    用 pip install 安装缺失的包     错误类型是 ConnectionError？
           ↓                          ↓ 是               ↓ 否
       重新运行                  确认网络是否正常     错误类型是 Unauthorized？
           ↓                     • 国内供应商：检查       ↓ 是            ↓ 否
       问题解决？                  有没有开不必要的     检查 Key 是否     错误信息中有
       ↓ 否         ↓ 是         代理（关了试试）      正确设置：       "rate" 或
    包的版本可能    🎉完成        • 国外供应商：          • 是否完整复制  "limit"？
    不兼容，尝试                  确认是否可以           • 环境变量名是     ↓ 是
    pip install                  科学上网              否正确          等待后重试
    <包>==指定版本                     ↓                • Key 是否过期    或换 Key
                                  问题解决？       ↓          ↓

```
