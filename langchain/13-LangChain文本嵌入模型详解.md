## 二、什么是文本嵌入

### 2.1 定义

> **文本嵌入（Text Embedding）**：将文本信息（单词、句子、段落）转换成**固定长度的数值向量**。这个向量由一系列数字组成，捕捉了文本的语义信息。

```
输入：自然语言文本
    ↓ Embedding 模型处理
输出：固定长度的数值向量

举例：
"产品太棒了" → [0.023, -0.451, 0.782, 0.134, ..., -0.567]
                ↑ 一个包含 1024 个数字的向量（以 embedding-2 为例）
```

### 2.2 直观理解

把 Embedding 想象成**翻译官**——把人类语言翻译成计算机能理解的数字语言：

```
┌─────────────────────────────────────────────────────────┐
│                  文本嵌入 = 翻译官                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  人类语言                    计算机语言                    │
│  ┌──────────┐              ┌──────────────┐              │
│  │ "你好"    │  ──翻译──→  │ [0.1, 0.3, -0.2, ...] │      │
│  │ "Hello"  │  ──翻译──→  │ [0.1, 0.3, -0.2, ...] │      │
│  │          │              │  ↑ 相似的向量！       │      │
│  │ "再见"    │  ──翻译──→  │ [-0.5, 0.8, 0.1, ...] │      │
│  │          │              │  ↑ 不同的向量         │      │
│  └──────────┘              └──────────────┘              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.3 核心特征——固定长度

无论输入文本多长或多短，输出的向量长度始终固定：

```
"好"                        → 1024 个数字
"这个产品真的太棒了！"         → 1024 个数字
"人工智能是计算机科学的一个分支..."（一整段）→ 1024 个数字

→ 同一个模型输出的向量维度始终相同
→ 这使得不同长度的文本可以在同一个向量空间中进行比较
```

## 三、为什么要转换成向量——计算机只懂数字

### 3.1 核心原因

> **计算机不认识字，但它会算数。**

```
你对计算机说："帮我打开这个文件夹"
计算机的反应：???（完全不理解）

你对计算机说：[0.23, -0.45, 0.78, ...] 
计算机的反应：收到！开始计算！
```

### 3.2 类比

```
就像一个只懂中文的人和一个只懂英文的人交流：

不懂英文的中国人：  "你好！" 
不懂中文的英国人：  "???"
                ↓ 需要一个翻译官
不懂英文的中国人：  "Hello!"（翻译后）
不懂中文的英国人：  "Hi there!"（懂了！）

计算机同理：
计算机：不懂"产品太棒了"
        ↓ Embedding 模型翻译
计算机：收到 [0.023, -0.451, ...]，可以开始计算了！
```

### 3.3 数字化的意义

一旦文本变成了数字向量，计算机就可以用**数学方法**来处理文本：

| 数学操作 | 对应的语义操作 |
|----------|---------------|
| 计算两个向量的**距离** | 判断两句话意思是否相近 |
| 计算两个向量的**余弦相似度** | 量化语义相似的程度 |
| 对向量做**聚类** | 把意思相近的文本分到一组 |
| 用向量做**分类** | 判断一段文本属于哪个类别 |

## 四、语义相似度——向量空间的核心规律

### 4.1 核心规律

> **语义相近的文本 → 向量空间中距离近、相似度高**
> **语义不同的文本 → 向量空间中距离远、相似度低**

### 4.2 实例一：语义相近 → 向量接近

```
"Hello there" 和 "Hi there" → 都是打招呼 → 语义相近

"Hello there" → [0.12, 0.45, -0.33, ...]
"Hi there"    → [0.13, 0.43, -0.31, ...]
                 ↑ 两组数字很接近！
                 
→ 相似度高 ✅
```

### 4.3 实例二：语义不同 → 向量远离

```
"What's your name" → 在提问
"My friends call me Word" → 在回答

"What's your name"        → [0.12, 0.45, -0.33, ...]
"My friends call me Word" → [-0.67, -0.21, 0.55, ...]
                              ↑ 两组数字差异很大！
                              
→ 相似度低 ✅
```

### 4.4 向量空间的可视化

如果把高维向量降维到 2D 空间来观察：

```
          语义相近的聚在一起
          
          好评区域                    差评区域
       ┌─────────────┐          ┌─────────────┐
       │  "太棒了"    │          │  "垃圾"      │
       │  "超好用"    │          │  "太差了"    │
       │  "很喜欢"  ● │          │        ● "烂" │
       │      ●      │          │  "失望"   ●  │
       │   ●  "推荐"  │          │              │
       └─────────────┘          └─────────────┘
       
       ↑ 相似度高的向量           ↑ 相似度高的向量
         互相靠近                   互相靠近
```

## 五、文本嵌入的四大应用场景

### 5.1 找相似内容——语义搜索

> 搜索时不仅匹配关键词，还能匹配**意思相近**的内容。

```
传统搜索：搜"勾股定理" → 只返回包含"勾股定理"这个词的文档

语义搜索：搜"勾股定理" → 还能返回包含"毕达哥拉斯定理"
                       "直角三角形边长关系"等意思相近的内容
```

| 搜索方式 | 搜"勾股定理"的结果 |
|----------|-------------------|
| 关键词匹配 | 只有包含"勾股定理"四个字的文档 |
| **语义搜索** | "勾股定理"+"毕达哥拉斯定理"+"直角三角形公式"等所有相关内容 |

### 5.2 分类文字——情感分析

> 判断一段评论是好评还是差评。

```
"产品太棒了，非常好用！"    → Embedding → 向量接近"好评"
"一般般吧，没什么特别的"     → Embedding → 向量接近"中评"
"垃圾，一天就坏了，退货！"   → Embedding → 向量接近"差评"

→ 同样的意思，不同的表达方式 → 向量都接近 → 能被正确分类
```

视频中提到：中文博大精深，夸人的方式有很多种，但"都是夸"意味着语义相近，转换成向量后距离也近。

### 5.3 推荐系统——基于内容的推荐

> 根据用户喜欢的内容，推荐语义相似的其他内容。

```
用户喜欢 → 文章A（关于Python机器学习）
            ↓ Embedding
         向量A
            ↓ 计算相似度
         向量B（关于Python深度学习）→ 相似！→ 推荐 ✅
         向量C（关于日本旅游）      → 不相似 → 不推荐 ❌
```

### 5.4 跨语言匹配

> 不同语言表达的相同意思 → 向量也相似。

```
中文："你好"
英文："Hello"
法文："Bonjour"

→ 三句话意思相同
→ Embedding 后向量非常接近
→ 不需要翻译就能知道它们是同一个意思！
```

## 六、选择 Embedding 模型

### 6.1 去哪儿找模型？

以智谱AI 平台为例：

```
智谱AI 开放平台
    ↓
找到「模型」或「大模型」入口
    ↓
找到「向量」或「Embedding」分类
    ↓
看到可用的 Embedding 模型
```

### 6.2 可用的模型版本

智谱AI 提供多个 Embedding 模型：

| 模型 | 向量维度 | 说明 |
|------|:------:|------|
| `embedding-2` | 256 / 512 / 1024 | 第二代模型，多维度可选 |
| `embedding-3` | 2048 等 | 第三代模型，维度更高 |

### 6.3 为什么维度很重要？

> **维度 = 向量里有多少个数字**

| 维度 | 含义 | 影响 |
|:----:|------|------|
| **256** | 向量有 256 个数字 | 信息量较少，但计算快 |
| **512** | 向量有 512 个数字 | 平衡 |
| **1024** | 向量有 1024 个数字 | 信息量丰富，计算稍慢 |
| **2048** | 向量有 2048 个数字 | 信息量最丰富 |

> ⚠️ **维度在后期数据匹配时会用到**，不同维度之间不能直接比较。选择模型时要注意维度的一致性。

### 7.1 依赖安装

### 7.2 完整代码——文本转 Embedding

```python
import os
from langchain_community.embeddings import ZhipuAIEmbeddings

# ============================================================
# 1. 设置 API Key
# ============================================================
os.environ["ZHIPUAI_API_KEY"] = "你的Key"

# ============================================================
# 2. 加载 Embedding 模型
# ============================================================
embeddings = ZhipuAIEmbeddings(
    model="embedding-2"   # 使用第二代 Embedding 模型
)
# 注意：要使用 embedding 模型，不是 chat 模型！

# ============================================================
# 3. 准备文本
# ============================================================
comments = [
    "产品太棒了！非常好用！",          # 好评
    "垃圾，一天就坏了，直接退货！"      # 差评
]

# ============================================================
# 4. 将文本转换为向量
# ============================================================
vectors = embeddings.embed_documents(comments)

# ============================================================
# 5. 查看结果
# ============================================================
# 查看向量维度
print(f"向量维度：{len(vectors[0])}")   # → 1024（以 embedding-2 为例）

# 查看第一个文本的向量（前10个数字）
print(f"好评向量（前10个）：{vectors[0][:10]}")

# 查看第二个文本的向量（前10个数字）
print(f"差评向量（前10个）：{vectors[1][:10]}")
```

### 7.3 运行结果

```
向量维度：1024

好评向量（前10个）：[0.0023, -0.0451, 0.0782, 0.0134, -0.0067, 0.0238, ...]
差评向量（前10个）：[-0.0523, 0.0192, -0.0331, 0.0789, 0.0012, -0.0437, ...]

→ 两组数字明显不同！
→ 我们不用看懂每一个数字，计算机知道怎么算就可以
```

## 八、代码实战——好评差评分类

### 8.1 逻辑回归简介

> **逻辑回归（Logistic Regression）** 是一种简单的分类算法。这里用它来判断新文本是好评还是差评。

工作流程：

```
第1步：准备训练数据
    ├── 好评文本 → Embedding → 向量A  ─┐
    └── 差评文本 → Embedding → 向量B  ─┤
                                       ↓
第2步：给向量打标签                       逻辑回归模型学习
    ├── 向量A → 标签 1（好评）             "好评的向量长这样"
    └── 向量B → 标签 0（差评）             "差评的向量长那样"
                                       ↓
第3步：预测新文本
    新文本 → Embedding → 向量C → 模型判断 → 1（好评）或0（差评）
```

### 8.2 依赖安装

```bash
# 安装 scikit-learn（用于逻辑回归）
pip install scikit-learn==1.7.1
```

### 8.3 完整代码

```python
import os
from langchain_community.embeddings import ZhipuAIEmbeddings
from sklearn.linear_model import LogisticRegression

# ============================================================
# 1. 准备模型
# ============================================================
os.environ["ZHIPUAI_API_KEY"] = "你的Key"
embeddings = ZhipuAIEmbeddings(model="embedding-2")

# ============================================================
# 2. 准备训练数据
# ============================================================
# 训练文本
comments = [
    "产品太棒了！非常好用！",          # 好评
    "垃圾，一天就坏了，直接退货！"      # 差评
]

# 将文本转换为向量
vectors = embeddings.embed_documents(comments)

# 标签：1 = 好评，0 = 差评
labels = [1, 0]

# ============================================================
# 3. 训练逻辑回归模型
# ============================================================
clf = LogisticRegression()
clf.fit(vectors, labels)    # 用向量和标签训练模型

# ============================================================
# 4. 准备测试文本
# ============================================================
test_comment = "这个东西超好用！"

# 将测试文本转换为向量
test_vector = embeddings.embed_documents([test_comment])

# ============================================================
# 5. 预测
# ============================================================
prediction = clf.predict(test_vector)[0]

# ============================================================
# 6. 输出结果
# ============================================================
if prediction == 1:
    print(f'"{test_comment}" → 好评 ✅')
else:
    print(f'"{test_comment}" → 差评 ❌')
```

### 8.4 运行结果

```
"这个东西超好用！" → 好评 ✅

为什么？因为：
├── "产品太棒了"（训练中的好评）→ 向量很接近"超好用"
├── "垃圾"（训练中的差评）→ 向量远离"超好用"
└── 模型判断：接近好评 → 输出 1 → 好评！
```

### 8.5 代码逐行解析

| 代码 | 作用 |
|------|------|
| `clf = LogisticRegression()` | 创建一个逻辑回归分类器 |
| `clf.fit(vectors, labels)` | 用训练数据"教会"分类器：这些向量是好/差评 |
| `clf.predict(test_vector)` | 对新向量做预测，返回 1（好评）或 0（差评） |
| `prediction[0]` | `predict` 返回数组，取第一个（也是唯一一个）结果 |

## 九、完整代码汇总

```python
import os
from langchain_community.embeddings import ZhipuAIEmbeddings
from sklearn.linear_model import LogisticRegression

# ============================================================
# 1. 设置和加载模型
# ============================================================
os.environ["ZHIPUAI_API_KEY"] = "你的Key"
embeddings = ZhipuAIEmbeddings(model="embedding-2")

# ============================================================
# 2. 准备训练数据
# ============================================================
train_texts = [
    "产品太棒了！非常好用！",
    "垃圾，一天就坏了，直接退货！"
]
train_vectors = embeddings.embed_documents(train_texts)
train_labels = [1, 0]   # 1=好评, 0=差评

# ============================================================
# 3. 训练分类器
# ============================================================
clf = LogisticRegression()
clf.fit(train_vectors, train_labels)

# ============================================================
# 4. 定义预测函数
# ============================================================
def predict_sentiment(text):
    """预测文本是好评还是差评"""
    vec = embeddings.embed_documents([text])
    result = clf.predict(vec)[0]
    return "好评 ✅" if result == 1 else "差评 ❌"

# ============================================================
# 5. 测试
# ============================================================
test_samples = [
    "这个东西超好用！",
    "太失望了，完全不值这个价",
    "推荐购买，质量很好",
    "一般般，没什么特别的"
]

for text in test_samples:
    print(f'"{text}" → {predict_sentiment(text)}')
```

**运行结果：**

```
"这个东西超好用！" → 好评 ✅
"太失望了，完全不值这个价" → 差评 ❌
"推荐购买，质量很好" → 好评 ✅
"一般般，没什么特别的" → 差评 ❌（中评更接近差评）
```

## 十二、课程小结

### 12.1 本课核心知识图谱

```
┌────────────────────────────────────────────────────────────┐
│               文本嵌入核心知识体系                            │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  1️⃣  什么是文本嵌入？                                       │
│      ├── 将文本转换成固定长度的数值向量                       │
│      ├── 输入：任意长度的文本                                │
│      └── 输出：固定维度的数字列表（如 1024 个浮点数）          │
│                                                            │
│  2️⃣  为什么要转换成向量？                                    │
│      ├── 计算机不认识字，但懂数字                             │
│      ├── 文本 → 数字 → 计算机可以计算                        │
│      └── 计算距离 = 判断语义相似度                           │
│                                                            │
│  3️⃣  语义相似度规律                                          │
│      ├── 语义相近 → 向量空间距离近 → 相似度高                 │
│      └── 语义不同 → 向量空间距离远 → 相似度低                 │
│                                                            │
│  4️⃣  四大应用场景                                            │
│      ├── 语义搜索：找意思相近的内容                           │
│      ├── 文本分类：好评/差评                                 │
│      ├── 推荐系统：推荐相似内容                               │
│      └── 跨语言匹配：不同语言相同意思                         │
│                                                            │
│  5️⃣  代码三步走                                              │
│      ├── embeddings = ZhipuAIEmbeddings(model="...")       │
│      ├── vectors = embeddings.embed_documents(texts)       │
│      └── clf.fit(vectors, labels) → clf.predict(new_vec)  │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 12.2 一句话总结

> **文本嵌入 = 把文字翻译成数字。** 计算机不懂"好"和"坏"的文字含义，但能算出 `[0.23, -0.45, ...]` 和 `[-0.67, 0.12, ...]` 的差异。语义相近的文本 → 向量相近 → 计算机就能"理解"语义。

### 12.3 练习题答案

> **第一题**：文本嵌入模型的定义是什么？
>
> - A. 将文本转换成图片格式 ❌（Embedding 输出的是向量，不是图片）
> - **B. 将文本转换成固定长度的数值向量，捕捉语义信息 ✅**
> - C. 将文本直接翻译成另一种语言 ❌（翻译是另一回事）
> - D. 将文本存储到数据库中的关键词集合 ❌（和数据库无关）
>
> **答案：B**
>
> **第二题**：文本嵌入的主要用途之一是？
>
> - **A. 查找语义相似的文本内容 ✅**（语义搜索是核心用途）
> - B. 直接生成新的文本内容 ❌（Embedding 不生成文本）
> - C. 将文本转换成语音 ❌（那是 TTS，不是 Embedding）
> - D. 优化数据库查询 ❌（无关）
>
> **答案：A**
