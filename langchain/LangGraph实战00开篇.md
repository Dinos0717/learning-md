## 一、课程背景——为什么需要 LangGraph

### 1.1 AI 应用的演进趋势

```
传统 AI 应用                            未来 AI 应用
┌─────────────────────┐          ┌─────────────────────────┐
│ 无状态               │          │ 持久化智能体              │
│ 一次性问答            │   ──→   │ 多轮复杂任务              │
│ 无记忆               │          │ 短期+长期记忆             │
│ 被动响应             │          │ 自主推理与执行            │
│ 单一模型调用          │          │ 多工具协同编排            │
└─────────────────────┘          └─────────────────────────┘
```

未来 AI 应用的四个关键词：

| 关键词 | 含义 | LangGraph 如何支持 |
|--------|------|-------------------|
| **记忆** | 能记住之前的对话和上下文 | 短期记忆 + 长期记忆管理 |
| **推理** | 能分析问题、制定计划 | 图结构编排，条件分支 |
| **自主** | 能自己做决策和行动 | Agent 自主调用工具 |
| **持久化** | 状态可以保存和恢复 | Durable Execution |

### 1.2 传统方式的局限

在之前的 LangChain 课程中，我们创建的 Agent 虽然能调用工具，但存在一些根本问题：

| 局限 | 表现 | 影响 |
|------|------|------|
| 无状态 | 每次调用是独立的，重启后全部丢失 | 无法支持长时间运行的任务 |
| 无持久化 | 执行过程中断后无法恢复 | 复杂任务中途崩溃 = 重头再来 |
| 编排能力弱 | 只能线性调用工具 | 无法实现复杂的分支、循环、条件逻辑 |
| 不可观察 | 执行过程是黑盒 | 调试困难，不知道哪一步出错 |

### 1.3 LangGraph 的定位

> **LangGraph 是一个专为持久化智能体设计的低层级编排框架与运行时。** 它不是 LangChain 的替代品，而是 LangChain 生态的底层补充——LangChain 让你方便地调用模型，LangGraph 让你精确地编排 Agent 的执行流程。

## 二、什么是 LangGraph——低层级编排框架

### 2.1 "低层级编排框架"的含义

```python
# ── LangChain（高层级）：封装好，开箱即用 ──
from langchain.agents import create_react_agent
agent = create_react_agent(llm, tools, prompt)
result = agent_executor.invoke({"input": "..."})
# ↑ 几行代码搞定，但底层细节不可控

# ── LangGraph（低层级）：从底层构建，自由度更高 ──
from langgraph.graph import StateGraph
graph = StateGraph(State)
graph.add_node("agent", agent_node)
graph.add_node("tools", tool_node)
graph.add_edge("agent", "tools")
graph.add_conditional_edges("tools", router, {"agent": "agent", "end": END})
# ↑ 每个节点、每条边、每个条件都自己定义——完全掌控
```

| 层级 | 代表框架 | 特点 | 自由度 | 上手难度 |
|------|:------:|------|:-----:|:------:|
| **高层级** | LangChain Agent | 封装好，方便快捷 | ⭐⭐ | ⭐ 容易 |
| **低层级** | LangGraph | 从底层编排，完全掌控 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ 需要学习 |

### 2.2 "编排框架"的含义

> **编排（Orchestration）**：定义多个组件之间的执行顺序、条件分支、循环、并行等复杂流程。

```
简单的 Agent 调用（LangChain 已足够）：
用户输入 → LLM 推理 → 调用工具 → 返回结果

复杂的 Agent 编排（需要 LangGraph）：
用户输入 → LLM 分析意图
              ↓
         ┌────┴────┐
         ↓         ↓
      查天气    查订单
         ↓         ↓
         └────┬────┘
              ↓
         LLM 综合两个结果
              ↓
         ┌────┴────┐
         ↓         ↓
      需要确认？  直接回答
         ↓         ↓
      Human介入   返回用户
```

### 2.3 LangGraph 版本说明

本系列课程基于 **LangGraph v1.0.5**，这是目前（2026年6月）的最新稳定版本。

## 三、LangGraph 的五大核心特性

### 3.1 特性一：持久化执行（Durable Execution）

> **Durable Execution** 是 LangGraph 最具标志性的特性。它不仅能保存状态，更能保存**执行上下文**。当 Agent 执行过程中出现故障时，可以从断点**无缝恢复**，确保任务永不中断。

```
一个复杂 Agent 任务的执行过程：

节点1（查天气）→ 完成 ✅
节点2（查订单）→ 完成 ✅
节点3（复杂计算）→ 执行到一半 → 💥 服务器重启！
    ↓
没有 LangGraph：从头再来，节点1和节点2白做了
有 LangGraph：从节点3的断点继续执行！节点1和2的结果保留！
```

| 能力 | 说明 |
|------|------|
| 状态保存 | 每执行完一个节点，状态自动持久化 |
| 故障恢复 | 崩溃后自动从上一个成功的节点继续 |
| 上下文保留 | 之前节点产生的中间结果不会丢失 |

### 3.2 特性二：完整的内存管理

> LangGraph 融合了**短期记忆**和**长期记忆**，为智能体赋予真正的**上下文感知能力**。

```
短期记忆（Short-term Memory）：
├── 当前对话的上下文
├── 本轮任务执行的中间状态
└── 生命周期：单次会话内

长期记忆（Long-term Memory）：
├── 跨会话的用户偏好
├── 历史交互的关键信息
└── 生命周期：持久化存储，跨会话
```

### 3.3 特性三：Human-in-the-Loop（人工介入）

> 在 Agent 执行的关键节点，允许**人类介入**进行确认、审核或干预。

```
应用场景：
├── 金融交易 → 大额转账前需人工确认
├── 内容发布 → AI生成的文章需人工审核
├── 医疗建议 → AI诊断需医生复核
└── 代码部署 → AI生成的代码需人工review
```

### 3.4 特性四：生产级可靠性

> LangGraph 已被**多家知名公司**用于生产环境。它不是实验性框架，而是经过实战验证的工业级工具。

### 3.5 特性五：深度可观察性（LangSmith 集成）

> LangGraph 与 **LangSmith** 深度集成，Agent 执行的每一步都会被监控和记录。

```
LangSmith 提供：
├── 每一步执行的输入/输出追踪
├── Token 用量统计
├── 延迟分析
├── 错误定位
└── LangGraph Studio → 可视化开发与调试
```

## 四、LangChain vs LangGraph——高层级 vs 低层级

### 4.1 全面对比

| 对比维度 | LangChain | LangGraph |
|----------|:--------:|:---------:|
| **定位** | 高层级 AI 应用框架 | 低层级 Agent 编排框架 |
| **抽象层级** | 高（封装好，开箱即用） | 低（从底层构建，精细控制） |
| **上手难度** | ⭐ 容易 | ⭐⭐⭐ 需要学习概念 |
| **灵活性** | ⭐⭐ 受限于封装 | ⭐⭐⭐⭐⭐ 完全自由 |
| **状态管理** | 基本 | 强大（持久化 + 恢复） |
| **流程控制** | 线性为主 | 图结构（分支/循环/并行/条件） |
| **适用场景** | 快速原型、简单 Agent | 复杂 Agent、生产环境 |
| **人机协作** | 不支持 | ✅ Human-in-the-Loop |
| **可观察性** | 有限 | ✅ LangSmith 全链路 |
| **关系** | 可以被 LangGraph 调用 | 可以编排 LangChain 组件 |

### 4.2 不是替代，是互补

> **LangGraph 不是 LangChain 的替代品，而是底层补充。** 在 LangGraph 中仍然可以使用 LangChain 的 ChatModel、Tool、PromptTemplate 等组件。LangGraph 负责"怎么编排"，LangChain 负责"具体做什么"。

## 六、LangSmith——全链路监控与可视化

### 6.1 LangSmith 的作用

> **LangSmith** 是 LangGraph 生态中的监控和调试平台。它与 LangGraph 深度集成，能追踪 Agent 执行的每一步。

```
LangSmith 提供的能力：
┌─────────────────────────────────────────┐
│                                         │
│  📊 执行追踪   每一步的输入/输出/Tokens   │
│  🔍 调试定位   快速找到出错节点          │
│  📈 性能分析   延迟、吞吐量统计          │
│  🎨 可视化     流程图 + 实时状态        │
│  🚀 一键部署   从开发环境到生产环境      │
│                                         │
└─────────────────────────────────────────┘
```

### 6.2 LangGraph Studio

> **LangGraph Studio** 已集成到 LangSmith 中，提供**可视化**的开发体验：

- 以图形方式查看和编辑 Agent 流程
- 实时查看每个节点的执行状态
- 拖拽式调整图结构
- 直接在 Studio 中测试和调试
