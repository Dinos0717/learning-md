# _TEMPLATE.md — Chunk 文件标准模板

> **Stage 1.5 使用此模板为每个模块创建 `_chunks/M0X-[模块名].md`。**
>
> 子 Agent（撰写者/审查者）读取 chunk 时，按此结构解析。

---

## 格式规范

```markdown
# M0X: [模块标题]

> 来源录音时间段: [HH:MM:SS] - [HH:MM:SS] (~X 分钟)
> 模块类型: [概念讲解 / 代码实操 / 案例分析 / Q&A答疑]
> 目标行数: ~XXX 行
> 对应 template.md 模板: [7.1 概念讲解型 / 7.2 代码实操型 / 7.3 案例分析型 / 7.4 Q&A答疑型]
> 状态: ⏳ 待撰写
> 创建时间: YYYY-MM-DD HH:MM

---
```

## 字段说明

| 字段 | 谁填 | 何时填 | 说明 |
|---|---|---|---|
| 模块元数据（标题、时间、类型等） | 主 Agent | Stage 1.5 | 从模块拆解结果填入 |
| 原始录音稿段落 | 主 Agent | Stage 1.5 | 从完整录音稿中截取粘贴 |
| 上下文提示 | 主 Agent | Stage 1.5 | 分析模块间关系后填入 |
| 撰写提示词 | 主 Agent | Stage 1.5 | 根据模块内容定制 |
| 撰写结果 | 子 Agent | Stage 2 | 追加在 `---` 之后 |

---

## 类型与模板对应

| 模块类型 | template.md 模板 | 推荐的小节结构 |
|---|---|---|
| 概念讲解 | 7.1 概念讲解型 | 直观理解 → 详细解释 → 对比辨析 → 常见误解 → 代码示例 |
| 代码实操 | 7.2 代码实操型 | 目标 → 前置准备 → Step-by-Step → 验证结果 → 排错指南 |
| 案例分析 | 7.3 案例分析型 | 背景 → 问题 → 分析过程 → 解决方案 → 结果与复盘 → 迁移启示 |
| Q&A答疑 | 7.4 Q&A答疑型 | 每个问题：原文 → 背景 → 回答 → 延伸 |
