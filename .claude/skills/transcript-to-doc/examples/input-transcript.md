# 示例输入：模拟录音稿

> 以下是一段 ~5 分钟的课程录音片段（S 级），用于演示 skill 的输入格式。

---

[00:00] 好，今天我们来讲一下 Python 里面的装饰器，decorator。这个东西很多初学者会觉得特别绕，但其实它本质上就是……嗯……一个函数，接收一个函数，返回一个新的函数。就这么简单。

[01:30] 我给大家写个例子啊。比如说我们现在有一个函数，用来打印"Hello"……

```python
def greet():
    print("Hello")
```

[02:15] 现在我想在不改 greet 函数的前提下，在它执行前后各打印一行日志。怎么做？哎对，装饰器。我们写一个 log_decorator……注意啊，这里很容易写错，我刚开始学的时候也踩过坑，把 wrapper 的括号写漏了，然后调了半天……

[03:00]（敲代码中）看，这样写：

```python
def log_decorator(func):
    def wrapper():
        print(f"开始执行 {func.__name__}")
        func()
        print(f"{func.__name__} 执行完毕")
    return wrapper
```

[04:00] 然后呢，用那个 @ 语法糖，@log_decorator 放在 greet 上面……大家注意，@log_decorator 就等价于 greet = log_decorator(greet)，这个一定要理解，不然后面带参数的装饰器你会完全懵掉。

[04:45] 好，有同学问"装饰器和 AOP 有什么关系"——好问题。装饰器其实就是 Python 实现 AOP（面向切面编程）的方式。AOP 的核心思想是把横切关注点——像日志、权限、缓存这些——从业务逻辑里分离出来。装饰器天然就适合干这个。如果你以后学 Spring 框架，会发现它的 AOP 和 Python 装饰器思路一模一样，只是写法不同。

[05:30] OK 那今天就讲这么多，大家下去之后可以试着写一个计时装饰器，统计函数的执行时间。能跑通就算掌握。
